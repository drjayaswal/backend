This is The Linux Branch

ls - listing files in current directory
pwd - showing present working directory
touch <file-name> - creating a file with the provided name
cat <file-name> - showing the content of file
mkdir <folder-name> - creating a folder with the provided name
rm <file-name> - remove/delete the file
rmdir <folder-name> - remove/delete the folder
head <file-name> - show starting line(s) of the file
tail <file-name> - show trailing line(s) of the file
tail -f <file-name> - show current and upcoming trailing line(s) of the file
less <file-name> - showing the content of file via pagination 
cp <file-name> <folder-name> - copy file with the provided name to the provided folder
mv <file-name> <folder-name> - copy/move file with the provided name to the provided folder and also remove the file from the source destination
wc <file-name> - showing count of Line(s) | Word(s) | Byte(s) respectively
<returning-command> | pbcopy - copying the terminal output to clipboard (macOS aka Darwin)
ln -s <fileToBeLink-name> <fileToLink-name> - creates a soft-link of --leftFile with ++rightFile
ln <fileToBeLink-name> <fileToLink-name> - creates a hard-link of --leftFile with ++rightFile
cut < -b for byte> < -c for character> <N - M range> <file-name> - cut the byte/character within the range i.e. N to M from file
<returning-command | tee <file-name> - creating a file with the content taken from the terminal output and shows it on terminal as well
sort <fileToBeSort-name> <newFile-name> - sort every line on basis of first character of the line and stores that sorted data into the provided file
diff <leftFile> <rightFile> - showing the difference with seperate colors between --left file and ++right file
vi <file-name> - editing the file via vim editor in the terminal
grep '<pattern/word>' <file-name> - showing the avaialblity of the pattern/word in the file
df/df -h - showing the details about the disk in a human-readable form with '-h' flag
ps - showing all the active processes
vailablity, and idu - showing the details about the disk usage
top - showing all running tasks
free - showing the memory space about the disk in a human-readable form with '-h' flag, and in human-readable 
kill -15 or -9 <PID> - killing the running process with PID with grace '-15' or force '-9' 
nohup <returning-command> - saving the terminal output from returning command to a 'nohup.out' file
vmstat - showing all details about memory including cache, buffer, etc using flags for active & inactive
uname - showing the base of platform
uptime - showing the activeness/login-span of the current machine
who - showing all available users on the machine
whoami - showing the current logged-in user on the machine
date - showing current date and time
which <software/utility> - showing the location of the provided software/utilitiy
id - showing all available user(s), group(s), etc with comma(,) seperation
sudo <permission-demanding-commands> - allowing the command to run with the help of 'sudo' allowance
apt install <software/utility> - installing software/utility from locally available resource
apt-get install <software/utility> - installing software/utility from resource available online
apt remove <software/utility> - removing/uninstalling software/utility from machine
useradd <user-name> - adding the user with provided name may requires 'sudo' command for permissions
su <user-name> - logging in with the provided user may requires 'sudo' command for permissions
passwrd <user-name> - changing password for the provided user may requies 'sudo' command for permissions
userdel <user-name> - removing the user with provided name may need 'sudo' command for permissions
groupadd <group-name> - creating a group with provided name may requires 'sudo' command for permissions
gpasswd <user-name> <group-name> - adding the user to the provided group may requires 'sudo' command for permissions, comes with '-a' flag for adding user(s) space seprated and '-M' flag for adding user(s) comman(,) seprated
